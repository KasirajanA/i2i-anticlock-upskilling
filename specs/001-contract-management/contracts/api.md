# API Contract: Contract Management

Base path: `/api/v1`
Auth: HTTP-only session cookie (`session_id`). All endpoints require an authenticated session.
All responses use `Content-Type: application/json` unless noted.

---

## Shared Error Schema

```json
{
  "detail": "Human-readable error message."
}
```

| HTTP Status | Meaning                                                     |
|-------------|-------------------------------------------------------------|
| 400         | Validation failure or business rule violation               |
| 401         | No valid session / session expired                          |
| 403         | Authenticated but insufficient role/ownership               |
| 404         | Resource not found                                          |
| 413         | Payload too large (file upload exceeds 1 MB)                |
| 422         | Pydantic schema validation error (auto-generated by FastAPI)|
| 500         | Unexpected server error                                     |

## Endpoints

### GET /api/v1/contracts

List contracts with optional filters. Paginated.

**Auth/Role**: All authenticated users. Sales Reps see all contracts (read-only on others' contracts is enforced at write endpoints).

**Query Parameters**:

| Parameter      | Type    | Required | Description                                      |
|----------------|---------|----------|--------------------------------------------------|
| status         | string  | No       | One of: Draft, Sent for Review, Active, Expired, Terminated |
| owner          | integer | No       | Filter by owner user ID                          |
| account_id     | integer | No       | Filter by account ID                             |
| end_date_from  | string  | No       | ISO 8601 date (YYYY-MM-DD), inclusive lower bound |
| end_date_to    | string  | No       | ISO 8601 date (YYYY-MM-DD), inclusive upper bound |
| page           | integer | No       | Default 1                                        |
| limit          | integer | No       | Default 20, max 100                              |

**Response 200**:

```json
{
  "total": 42,
  "page": 1,
  "limit": 20,
  "items": [
    {
      "id": 1,
      "ref_id": "CON-0001",
      "value": "12500.00",
      "start_date": "2026-01-01",
      "end_date": "2026-12-31",
      "status": "Active",
      "is_renewal_due": false,
      "is_archived": false,
      "owner_id": 5,
      "account_id": 3,
      "deal_id": null,
      "created_at": "2026-01-01T09:00:00Z",
      "updated_at": "2026-03-15T14:22:00Z"
    }
  ]
}
```

**Notable 4xx**: `400` if `end_date_from` > `end_date_to`.

### POST /api/v1/contracts

Create a new contract. Auto-generates `ref_id` (CON-NNNN). Initial status is always `Draft`.

**Auth/Role**: Sales Rep, Admin, Manager.

**Request Body**:

```json
{
  "value": 12500.00,
  "start_date": "2026-01-01",
  "end_date": "2026-12-31",
  "description": "Annual SaaS licence agreement",
  "account_id": 3,
  "deal_id": 7
}
```

`deal_id` is optional. `owner_id` defaults to the authenticated user. Admins and Managers may supply an explicit `owner_id`.

**Response 201**:

```json
{
  "id": 1,
  "ref_id": "CON-0001",
  "value": "12500.00",
  "start_date": "2026-01-01",
  "end_date": "2026-12-31",
  "status": "Draft",
  "description": "Annual SaaS licence agreement",
  "is_renewal_due": false,
  "is_archived": false,
  "owner_id": 5,
  "account_id": 3,
  "deal_id": 7,
  "created_at": "2026-06-16T08:00:00Z",
  "updated_at": "2026-06-16T08:00:00Z"
}
```

**Notable 4xx**: `400` if `start_date` > `end_date`; `404` if `account_id` or `deal_id` does not exist.

### GET /api/v1/contracts/{ref_id}

Fetch a single contract by its reference ID.

**Auth/Role**: All authenticated users.

**Path Parameter**: `ref_id` — e.g., `CON-0001`.

**Response 200**:

```json
{
  "id": 1,
  "ref_id": "CON-0001",
  "value": "12500.00",
  "start_date": "2026-01-01",
  "end_date": "2026-12-31",
  "status": "Active",
  "description": "Annual SaaS licence agreement",
  "is_renewal_due": false,
  "is_archived": false,
  "owner_id": 5,
  "account_id": 3,
  "deal_id": 7,
  "created_at": "2026-01-01T09:00:00Z",
  "updated_at": "2026-03-15T14:22:00Z",
  "attachment": {
    "id": 2,
    "filename": "signed_agreement.pdf",
    "mime_type": "application/pdf",
    "size_bytes": 204800,
    "created_at": "2026-01-10T11:00:00Z"
  }
}
```

`attachment` is `null` when no file has been uploaded.

**Notable 4xx**: `404` if `ref_id` not found.

### PATCH /api/v1/contracts/{ref_id}

Update editable fields of a contract. Only `Draft` and `Sent for Review` contracts may be edited via this endpoint; field edits to `Active`/`Expired`/`Terminated` contracts are blocked (use the transition endpoint for status changes).

**Auth/Role**: Sales Rep (own contracts only), Admin, Manager.

**Request Body** (all fields optional):

```json
{
  "value": 15000.00,
  "start_date": "2026-01-01",
  "end_date": "2027-01-01",
  "description": "Revised terms",
  "deal_id": 9
}
```

**Response 200**: Updated contract object (same schema as GET single).

**Notable 4xx**: `403` if Sales Rep attempts to edit a contract they do not own; `400` if attempting to edit a contract in `Active`, `Expired`, or `Terminated` status.

### POST /api/v1/contracts/{ref_id}/transition

Advance or revert the lifecycle status of a contract.

**Auth/Role**: Sales Rep (forward transitions on own contracts), Admin (all transitions including backward), Manager (forward only).

**Request Body**:

```json
{
  "status": "Active",
  "note": "Reviewed and approved by legal."
}
```

`note` is required for backward transitions (Admin only); optional for forward transitions and stored when provided.

**Response 200**:

```json
{
  "ref_id": "CON-0001",
  "previous_status": "Sent for Review",
  "new_status": "Active",
  "logged_at": "2026-06-16T10:30:00Z"
}
```

**Notable 4xx**:
- `400` — invalid transition (e.g., Draft → Active skipping Sent for Review).
- `400` — backward transition attempted without a note.
- `403` — non-Admin attempts a backward transition.
- `403` — Sales Rep attempts to transition a contract they do not own.

### POST /api/v1/contracts/{ref_id}/renew

Create a successor `Draft` contract pre-filled from the original. Dates are advanced by one term length (end_date - start_date). A `RenewalLink` record is created linking original to successor.

**Auth/Role**: Sales Rep (own contracts), Admin, Manager.

**Request Body**: Empty `{}` or omitted.

**Response 201**:

```json
{
  "original_ref_id": "CON-0001",
  "successor": {
    "id": 14,
    "ref_id": "CON-0014",
    "value": "12500.00",
    "start_date": "2027-01-01",
    "end_date": "2027-12-31",
    "status": "Draft",
    "account_id": 3,
    "deal_id": 7,
    "owner_id": 5,
    "created_at": "2026-06-16T10:35:00Z",
    "updated_at": "2026-06-16T10:35:00Z"
  }
}
```

**Notable 4xx**:
- `400` — contract already has a renewal (RenewalLink exists for this original).
- `400` — cannot renew a `Draft` or `Sent for Review` contract (must be `Active` or `Expired`).

### POST /api/v1/contracts/{ref_id}/attachment

Upload or replace the single file attachment for a contract. If an attachment already exists, it is deleted from disk and its metadata record is replaced.

**Auth/Role**: Sales Rep (own contracts), Admin, Manager.

**Request**: `multipart/form-data`

| Field | Type   | Required | Description                        |
|-------|--------|----------|------------------------------------|
| file  | binary | Yes      | PDF, DOCX, or image; max 1 MB      |

**Response 201**:

```json
{
  "id": 3,
  "contract_id": 1,
  "filename": "contract_v2.pdf",
  "mime_type": "application/pdf",
  "size_bytes": 512000,
  "created_at": "2026-06-16T11:00:00Z"
}
```

**Notable 4xx**:
- `413` — file exceeds 1 MB.
- `400` — unsupported MIME type (only `application/pdf`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, `image/jpeg`, `image/png` accepted).
- `403` — Sales Rep attempting to attach to a contract they do not own.

---

### DELETE /api/v1/contracts/{ref_id}/attachment

Remove the current file attachment from a contract. Deletes the file from disk and removes the metadata record.

**Auth/Role**: Sales Rep (own contracts), Admin, Manager.

**Response 204**: No body.

**Notable 4xx**: `404` if no attachment exists on the contract; `403` if Sales Rep does not own the contract.

---

### GET /api/v1/contracts/{ref_id}/activity

Retrieve the activity log for a contract, ordered newest-first.

**Auth/Role**: All authenticated users.

**Response 200**:

```json
{
  "ref_id": "CON-0001",
  "logs": [
    {
      "id": 5,
      "action_type": "status_transition",
      "actor_id": 5,
      "actor_name": "Jane Smith",
      "note": null,
      "created_at": "2026-03-15T14:22:00Z"
    },
    {
      "id": 4,
      "action_type": "attachment_added",
      "actor_id": 5,
      "actor_name": "Jane Smith",
      "note": "Uploaded signed PDF.",
      "created_at": "2026-02-01T09:10:00Z"
    }
  ]
}
```

**Notable 4xx**: `404` if `ref_id` not found.
